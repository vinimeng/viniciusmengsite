BetterStartUp

Pequeno programa feito em C# para gerenciar a inicialização de programas no sistema operacional Windows.

Funcionalidades:

  • Retirar programas da inicialização do Windows;
  • Adicionar programas a lista de inicialização do BetterStartUp;
  • Definir ordem dos programas adicionados a lista de inicialização;
  • Definir delay de inicialização dos programas adicionados a lista;
  • Definir parâmetros de linha de comando dos programas adicionados a lista;

LevelEditor

Editor de sprites arrasta e solta desenvolvido em Typescript. Tem o objetivo de ser o editor de níveis do jogo web que estou desenvolvendo.

Funcionalidades:

  • Permite que níveis sejam feitos no próprio navegador;
  • Salvamento do nível feito em arquivo JSON;
  • Carregamento de níveis salvos para continuar editando;
  • Autosalvamento (nos casos de sair sem salvar o progresso);

LaudosWeb

O LaudosWeb é um projeto de um RIS (Radiological information system) em linguagem web voltado para clinícas de diagnóstico por imagem.

Funcionalidades:

  • Integrações com ERPs;
  • Integrações com PACS;
  • Fluxo de atendimento de pacientes;
  • Gravação de áudios e digitação de laudos médicos;
  • Visualização de exames online;

O projeto não é de minha autoria, no momento sou apenas o principal desenvolvedor do projeto.

→ LaudosWeb da Clínica Vero Dellaudo

→ LaudosWeb da Clínica Sul Diagnóstico

FM Sistemas

O FM Sistemas é um sistema de gerenciamento de radiofármacos, voltado principalmente para o fluxo de trabalho de exames de medicina nuclear.

Funcionalidades:

  • Cadastros e controles de geradores, eluição, marcação, fracionamento, administração e evolução;
  • Entrada e saída de materiais;
  • Cadastros de pacientes, agendas e atendimentos;
  • Controle de acesso e rastreamento;
  • Relatório e gráficos;

O projeto não é de minha autoria, fui apenas um dos desenvolvedores do projeto.

→ FM Sistemas

Impressão Sob Demanda

O projeto consiste em dar a possibilidade para executar as impressões de exames DICOM sob demanda.

Funcionalidades:

  • Impressões vindas diretamente dos aparelhos médicos;
  • Rotina de OCR para obter informações diretamente das imagens (ID, Nome do paciente, Accession number);
  • Criação de regras específicas para cada aparelho médico para executar rotinas de OCR nas imagens;
  • Integração com worklist para vincular informações;
  • Possibilidade de integrações com sistemas existentes;

Sou o principal desenvolvedor do projeto.
